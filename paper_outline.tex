\subsection{Per-Zone Inference: Rolling Buffer and TTI}
% OUTLINE: The core "prediction" mechanism.
%
% Rolling Buffer:
% - Circular buffer of last N = buffer_seconds / dt samples per sensor node
% - Discards readings older than buffer_seconds
%
% Rate Estimator:
% - dT/dt estimated via ordinary least-squares fit on buffer contents
% - More robust than finite difference over noisy readings
%
% Time-to-Impact (TTI):
%   TTI = (T_sp - T) / (dT/dt)   if dT/dt > 0
%   TTI = \infty                  otherwise
%
% Urgency:
%   u = 1 / TTI   (higher u => zone is approaching setpoint faster)
%
% This is the AoI-aware signal: each zone computes its own urgency from
% current data (AoI = 0), with no dependence on a central poll.

\todo{Show TTI and urgency as display equations. One sentence each on
why OLS over the buffer is preferred to finite difference.}

\subsection{Urgency Gossip and Resource Negotiation}
% OUTLINE: Describe the gossip protocol and proportional allocation.
%
% Gossip Protocol:
% - Threshold-triggered: node broadcasts urgency only if |dT/dt| > talk_threshold
%   (avoids unnecessary communication when temperature is stable)
% - After gossip_rounds rounds, each zone aggregates neighborhood urgencies
%
% Proportional Damper Allocation (edge policy):
%   A_i = u_i / \sum_j u_j
%   Q_cool,i = A_i \times Q_total
%
% Centralized Baseline:
% - Central controller polls all sensors every polling_interval seconds
%   (+ Gaussian jitter \sigma, + compute delay)
% - Issues damper commands using stale urgency data
% - AoI = polling_interval + jitter + compute_delay (configurable)
%
% Both policies use the same proportional formula; only the data freshness
% differs. This isolates the effect of AoI on control performance.
